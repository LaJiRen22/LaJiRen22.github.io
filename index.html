编写CSS时，div元素中的文字和img视觉上不同高的问题及解决

发表于 2022-10-22

首先，附上相关的html代码，我把img元素放在了div里面。

原因分析

造成这种效果其实是因为下载的矢量图标，非透明的部分里图片的底部有一定元素，而默认两个div元素对齐方式为

这使得div和图片的透明底边对齐，所以在视觉上不同高度。

解决方案

所以，我用的解决方案就是改变两个div的对齐方式。同时，也取消float，而是将display设置为inline-block，将div转为行内元素，因为使用float之后，似乎不能通过margin和padding来改变元素的位置。

使用Android Studio过程中遇到的问题总结

发表于 2022-10-06 更新于 2022-10-26

最近要跑一个比较古老的安卓项目，结果在导入过程中遇到了很多报错，心态有点崩溃。所幸，在万能的Google帮助下，花了不少时间，最后成功地跑起来了。为了方便以后遇到报错能更快解决，同时也帮助遇到同样问题的朋友们，我写了一个问题总结。

1、Installed Build Tools revision 33.0.0 is corrupted. Remove and install again using the SDK Manager.

原因分析

sdk版本和avd（AS用来运行的模拟器，如手机）采用的不匹配，avd版本可以打开查看，我的目前是30

解决方法

参考网址https://blog.csdn.net/weixin_56193843/article/details/118938788

不要使用最新的Build-Tools，把最新的打包工具卸载了，转而使用其他版本的Build-Tools

  1. 设置SDK Manager（更改sdk版本）

  2. 设置build.gradle文件

2、Unrecognized Android Studio (or Android Support plugin for IntelliJ IDEA) version ‘202.7660.26.42.7486908’, please retry with version 2021.2.1 or newer.

原因分析

​ Gradle版本不匹配

解决方法

比如将其修改为 4.0.1

3、Could not determine java version from ‘11.0.8‘

原因分析

​ Gradle版本过低（经过上一个错误之后，我看了下项目的gradle.bulid文件，版本是4.0.1，然后把上图的Gradle Version也改成了4.0.1，结果反而导致版本太低）

解决方法

​ 实际上报错的信息有提示Minimum Version，即最低版本，只需在Project Structure里选择大于等于最低版本的即可，比如我选择的6.1.1

4、Your anti-virus program might be impacting your build performance

原因分析

​ 如提示所说，这是因为电脑的防火墙或者安装的杀毒软件（比如火绒）造成的。

解决方法

​ 如果是防火墙，就在Windows安全中心添加安全项，如果安装了火绒，则在信任区添加，如下图

但奇怪的是，我这样设置了以后仍然有这个提示。到现在还没有完全解决，但是似乎并不影响程序运行。

5、点击运行后没有出现虚拟机的界面

原因分析

​ 首先要保证配置好了要运行的手机。重启Android Studio没有效果上网查询原因后，是因为内存不够了。

解决方法

​ 使用快捷键Ctrl+Alt+Del,调出任务管理器，杀死一些暂时不需要用到的进程；

​ 或者有安卓手机的话，链接真机进行模拟。

6、AS编译报错：Caused by: org .gradle .api. internal. plugins. Plugin Application Exception: Failed to apply plugin [id ‘ com. android. application’]

原因分析

从日志第一句可以看出，应该是项目路径包含中文字符串

解决方法

把项目移动到没有中文字符的路径中，也可以通过添加android.overridePathCheck=true解决。
